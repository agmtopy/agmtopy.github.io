gradle的安装与使用

由于新的项目大多是使用gradle进行依赖管理，因此在这里将其用法做一下总结

安装

主要步骤是：

  1. 下载安装包
  2. 配置环境变量
  3. 查看版本号

    gradle -v

  • 显示对应版本号表示安装成功

基础知识

gradle是一个基于Ant和Maven的理念的项目自动化构建工具。基于Groovy的特定领域语言（DSL）来声明项目设置。

gradle与maven的对比

  1. 灵活性更高
  2. 性能更好
  3. 依赖管理

gradle构建生命周期

gradle构建系统的生命周期可以分为：初始化、配置、运行三个阶段

  1. 初始化阶段
    此阶段读取根目录下的setting.gradle中的include信息，来决定构建哪些工程。以spring为例
  1. 配置阶段
    此阶段主要解析每个项目中的build.gradle配置文件、处理依赖关系、执行顺序
    以spring beans和spring-core两个项目为例
  • spring-beans的gradle文件
  • spring-core的gradle文件
  1. 运行阶段

    根据Gradle命令传递过来的任务名称，执行相关依赖任务。此阶段真正构建项目并执行项目下的各个任务。Gradle就会将这个任务链上的所有任务全部按依赖顺序执行一遍。

gradle插件

task的概念

gradle是通过task任务的模型进行驱动开发的，在一个gradle工程中将不同的构建工作打散成不同的task。task脚本采用Groovy语言进行编写。

基础命令

  • build 编译命令

    ./gradlew build –scan

表示编译项目并上传相应的编译数据到云端进行分析，本地分析的命令是

./gradlew clean app:assembleDebug –profile
该命令会在build/report中生成对应的报告

  • gradle build

    gradle build
    编译（编译过程中会进行单元测试）

  • gradle test

    gradle test
    单元测试

  • gradle test

    gradle build -x test
    编译时跳过单元测试

  • gradle run

    gradle run
    直接运行项目

  • gradle clean

    gradle clean 清空所有编译、打包生成的文件(即：清空build目录)

build.gradle模板

总结

  • 查看task

    gradle task

  • 编译-编译过程中会进行单元测试

    gradle build

  • 单元测试

    gradle test

  • 编译时跳过单元测试

    gradle build -x test

  • 直接远行项目

    gradle run

  • 清空build文件

    gradle clean

  • 安装jar到maven仓库

    gradle install
